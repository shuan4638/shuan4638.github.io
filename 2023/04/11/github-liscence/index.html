GitHub Liscense

Posted on 2023-04-11 Views:

When browsing other’s code on GitHub, we often see LISCENSE file in the repostory. This is an article to simply explain why do we needLISCENSE and the comparison of four popular LISCENSE.

What and Why LISCENSE?

When people share codes on GitHub, these codes are basically public for everyone, meaning everyone can view, learn and use the codes for their purpose.

However, different projects have different restictions of how they want their codes to be used. These different retrictions are listed in the different LISCENCE. In short, you can think LISCENCE if pretty much similar to “the rule of using codes”

In general, when people copy codes from your repo, they must include the liscence you made. This allows you to protect your copyright.

Popular LISCENSEs

There are 3 popular LISCENCE to use, which I list them from the most opening to the most restricting:

1. MIT license (Massachusetts Institute of Technology)

If you want to share codes with the most usage freedom

MIT license allows almost any kind of usage only requiring preservation of copyright and license notice.

Permissions Conditions (usage rules) Limitations
Commercial use License and copyright notice Liability
Distribution Warranty
Modification
Private use

2. Apache License 2.0

If you want to keep the fidelity of your original contributions

People need to explicitly state their changes whenever they make. Big company usually use this liscence.

Permissions Conditions (usage rules) Limitations
Commercial use License and copyright notice Liability
Distribution State changes Warranty
Modification
Private use
Patent use

3. GNU General Public License (GNU GPL)

If you do not want others to use your code for their own benefit

The software program is free, but all modified/extended version of this program must also be free.

Permissions Conditions (usage rules) Limitations
Commercial use License and copyright notice Liability
Distribution State changes Warranty
Modification Disclose source
Private use Same license (library)
Patent use

What happen if we don’t use LISCENSE?

Sharing code without liscence means that you don’t allow other to either use or modify your codes by default.
